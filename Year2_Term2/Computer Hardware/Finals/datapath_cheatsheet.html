How to Solve Any Question — 4 Steps
1
Identify instruction type
Has X(Reg) offset AND loading a register? → LOAD
Has X(Reg) offset AND writing to memory? → STORE
Two register operands, result into a register? → ALU (ADD, XOR, AND…)
Saves return address? → CALL
2
Determine MuxB
Is there an immediate / offset in the instruction? → MuxB = 1
Are both operands registers? → MuxB = 0
Tip: LOAD and STORE always have an offset → always MuxB = 1
3
Compute RY after Stage 4
LOAD → RY = value read from memory at address (base + offset)
ALU → RY = result of ALU operation (use updated register values!)
STORE → RY = don't care (nothing written to register)
CALL → RY = PC + 4 (return address)
4
RF_write in Stage 5
STORE → RF_write = 0 (writes memory, NOT a register)
Everything else (LOAD, ALU, CALL) → RF_write = 1
⚡ Common Exam Traps
Register values UPDATE after each instruction. Instruction 2 uses the NEW value written by Instruction 1.
Store: RF_write = 0. Students always write 1 here. Store writes to MEMORY, not to a register.
For LOAD, RY = Mem[base + offset], NOT the base register value itself.
For ALU ops, RY = ALU result (RZ passed through MuxY=0). Not RM, not memory.
Store's RY is irrelevant — don't waste time computing it.
